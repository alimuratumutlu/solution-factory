# Religious Content Review Workflow

## Principle

Spiritual support is optional reflective support. It must not be presented as a
guaranteed way to create success, healing, income, marriage, reconciliation, or
any other real-world outcome.

Practical action steps remain the core product output. Esma, dua, and dhikr
content can only appear when the user enables spiritual support.

## Source Origin

Every spiritual practice must declare `source.origin`:

- `catalog`: imported from a known catalog such as `5-min-dhikr`
- `user_defined`: created or edited directly by the user
- `ai_suggested`: generated by AI and not yet accepted into a reviewed catalog

AI-suggested content should not be silently promoted into the reviewed catalog.
It needs an explicit review step first.

## Review Status

Allowed `source.reviewStatus` values:

- `needs_religious_review`: imported or suggested content awaiting review
- `reviewed`: checked and accepted for product use
- `rejected`: blocked from recommendation output
- `user_defined`: shown only as the user's own saved practice
- `ai_suggested`: draft AI content, not catalog truth

Current imported catalog records are marked `needs_religious_review`.

## AI Guardrails

AI may:

- match a user's confirmed context to existing reviewed or pending catalog
  intention tags
- explain why a practice was suggested as reflective support
- preserve catalog day, count, transliteration, and source metadata
- apply the user's preferred reminder time when no exact source time exists

AI must not:

- invent Arabic text, counts, religious attribution, or guaranteed benefits
- claim a practice will produce success, money, healing, marriage, pregnancy,
  reconciliation, legal wins, or safety
- replace medical, legal, financial, safety, or mental-health guidance
- hide whether content is reviewed, user-defined, or AI-suggested
- show spiritual content when the user disabled spiritual support

## High-Stakes Topics

When the user's problem touches health, mental health, legal, financial, safety,
or crisis topics, the app should keep spiritual support secondary and show
practical safety guidance. The plan should encourage qualified professional
help where appropriate.

## Recommendation Output

Recommendation output must include:

- `reason`
- `matchedIntentions`
- `schedule.days`
- `schedule.timingLabel`
- `schedule.count`
- `schedule.timeOfDay`
- `source.origin`
- `source.path`
- `source.reviewStatus`

This lets the UI and any future API distinguish reviewed catalog content,
user-defined practices, and AI-suggested drafts.
